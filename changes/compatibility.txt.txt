Architectural Changes to the ONNX GraphAttribute Parsing Robustness: * Change: Added dynamic type-checking when reading the Einsum node's equation attribute.Reason: Depending on the onnx library version, attributes are loaded as either raw bytes or decoded str objects. The script safely unwraps the equation string (e.g., bchw,bnc->bnhw) regardless of the library version to identify the specific attention nodes that need replacing.Spatial Flattening (Reshape):Change: The script inserts a Reshape node to flatten the 4D input feature map ($X$) from [Batch, Channels, Height, Width] into a 3D tensor of shape [Batch, Channels, Height * Width].Reason: Matrix multiplication (MatMul) requires 2D or 3D matrices. By flattening the spatial dimensions ($H$ and $W$) into a single sequence length, the tensor becomes compatible with a standard dot-product operation.Core Computation (MatMul):Change: Inserted a MatMul node that multiplies the query tensor ($Y$ with shape [Batch, Queries, Channels]) against the newly flattened spatial tensor.Reason: This performs the exact same mathematical contraction over the $C$ (Channel) dimension as the original Einsum node, resulting in an intermediate tensor of shape [Batch, Queries, Height * Width].Dynamic Dimension Extraction (Shape & Slice):Change: Inserted Shape nodes to read the dimensions of the original input tensors at runtime. Slice nodes are then used to extract the [Height, Width] from the $X$ tensor and the [Batch, Queries] from the intermediate MatMul tensor.Reason: Hardcoding the resolution (e.g., exactly 384x384) would break the model if it needed to be run at different scales or non-square aspect ratios (like 16:9). Dynamic extraction ensures the patch adapts to whatever input resolution the model receives.Target Shape Formulation (Concat):Change: Added a Concat node to merge the extracted [Batch, Queries] and [Height, Width] dimensions into a single 1D shape tensor: [B, N, H, W].Reason: This dynamically builds the exact blueprint needed to un-flatten the data back into its proper spatial format.Final Spatial Restoration (Reshape):Change: A final Reshape node takes the output of the MatMul and applies the dynamically concatenated shape tensor, restoring the data to [Batch, Queries, Height, Width].Reason: This ensures the output of the new sub-graph matches the exact shape and data layout that the downstream layers of the RF-DETR model expect, keeping the rest of the architecture completely intact.Graph Reconnection and Cleanup:Change: The original Einsum node's outputs are disconnected, and the new sub-graph is spliced into the main network. A topological sort (toposort) is then executed.Reason: Topological sorting ensures all newly created nodes are executed in the correct sequential order, preventing runtime errors when QAIRT compiles the execution sequence for the DSP.
